# KlassFile DSL

## Overview

The **KlassFile DSL** is a Kotlin DSL for generating Java class files at runtime through the Java ClassFile API. It lets you describe a class structure declaratively—its name, flags, superclass, implemented interfaces, fields, methods, and bytecode instructions—while keeping the syntax Kotlin-friendly.

The public entry point is `KlassFileBuilder.klass(name) { ... }`, which returns a `Klass` wrapper around the generated bytes.

## Quick start

```kotlin
val generated = KlassFileBuilder.klass("com.example.Dog") {
	flags(ACC_PUBLIC)
	inherit<Animal>()

	val nameField = field<String>("name")

	constructor {
		defaultConstructor()
		putThisField(nameField, "Jerry")
		return_()
	}

	method("speak", returnType = CD_String) {
		ldc("Woof! My name is ")
		getThisField(nameField)
		invokevirtual<String>("concat", String::class)
		areturn()
	}
}

val dogClass = generated.writeAndGetClass<Animal>()
val dog = dogClass.createInstance()
```

## Main concepts

- `KlassFileBuilder` collects the class definition.
- `FieldRef<T>` and `MethodRef` let you keep references to generated members.
- `Klass` is the output wrapper that exposes the generated bytes and loading/writing helpers.
- `CodeBuilder` extension helpers are available inside method bodies to emit bytecode instructions.

## Class configuration

### Entry point: `klass(name, builder)`

```kotlin
val klass = KlassFileBuilder.klass("com.example.MyClass") {
	// class definition here
}
```

This helper:

- creates a new `KlassFileBuilder`
- applies the DSL block
- finalizes the class into a `Klass` wrapper

### `flags(value: Int)`

Sets the access flags of the generated class.

```kotlin
KlassFileBuilder.klass("com.example.MyClass") {
	flags(ACC_PUBLIC or ACC_FINAL)
}
```

Use bitwise OR to combine flags such as `ACC_PUBLIC`, `ACC_FINAL`, and `ACC_ABSTRACT`.

### `classDesc(...)`

The DSL provides several overloads for creating `ClassDesc` values:

- `classDesc(name: String)`
- `classDesc(clazz: Class<*>)`
- `classDesc(klass: KClass<*>)`
- `classDesc<T>()`

Examples:

```kotlin
val stringDesc = classDesc<String>()
val mapDesc = classDesc("java.util.HashMap")
```

Notes:

- `classDesc(name)` expects a **binary class name** such as `java.lang.String`.
- Primitive Kotlin/JVM types are mapped to predefined constants such as `CD_int` and `CD_void`.

### Superclass and interfaces

The hierarchy API provides both descriptor-based and generic reified overloads:

- `extends(classDesc: ClassDesc = CD_Object)` and `extends<T>()`
- `implements(classDesc: ClassDesc)` and `implements<T>()`

When a reified type is available, prefer the generic overloads.

#### `extends(classDesc: ClassDesc = CD_Object)`

Sets the superclass of the generated class.

```kotlin
KlassFileBuilder.klass("com.example.Child") {
	extends<SomeBase>()
}
```

#### `implements(classDesc: ClassDesc)`

Adds an interface to the list of implemented interfaces.

```kotlin
KlassFileBuilder.klass("com.example.Child") {
	implements<Serializable>()
}
```

#### `inherit<T>()`

Convenience helper that chooses between `implements<T>()` and `extends<T>()`:

- if `T` is an interface, it is added to the implemented interfaces
- if `T` is an abstract class, it becomes the superclass
- otherwise, an `IllegalArgumentException` is thrown

```kotlin
KlassFileBuilder.klass("com.example.Child") {
	inherit<Serializable>()
}
```

## Fields

### `field(name, type, flags)`

Defines a field using an explicit `ClassDesc` and returns a `FieldRef<T>` that can later be used inside method bodies.

This is the lower-level overload; if you have a reified type available, prefer `field<T>(name, flags)`.

```kotlin
val nameField = field<String>("name", CD_String)
val counterField = field<Int>("counter", CD_int, ACC_PRIVATE or ACC_STATIC)
```

Behavior:

- default access is `ACC_PRIVATE`
- the returned `FieldRef` refers to the field in the generated class
- the `type` argument is a `ClassDesc` describing the field type

### `field<T>(name, flags)`

Same as `field(name, type, flags)`, but infers the field type from `T`.

```kotlin
val active = field<Boolean>("active", ACC_PRIVATE)
```

### `publicField<T>(name)`

Defines a public field.

```kotlin
val id = publicField<Long>("id")
```

### `staticField<T>(name)`

Defines a static field.

```kotlin
val version = staticField<String>("version")
```

## Methods

All method builders return a `MethodRef`. That reference can be stored and reused later when generating invocation bytecode.

### `constructor(flags = ACC_PUBLIC, vararg args, body)`

Defines a constructor.

```kotlin
constructor {
	defaultConstructor()
	return_()
}
```

Important details:

- the default access is `ACC_PUBLIC`
- the default body calls the superclass constructor and returns
- if the `ACC_STATIC` flag is supplied, the generated member is named `<clinit>` instead of `<init>`

### `method(name, flags = ACC_PUBLIC, returnType = CD_void, vararg args, body)`

Defines a general method.

```kotlin
val speak = method("speak", returnType = CD_String) {
	ldc("Hello")
	areturn()
}
```

### `staticMethod(name, returnType = CD_void, vararg params, body)`

Convenience wrapper for a public static method.

```kotlin
staticMethod("create", classDesc<MyType>()) {
	// emit bytecode
}
```

### `privateMethod(name, returnType = CD_void, vararg params, body)`

Convenience wrapper for a private method.

### `main(body)`

Defines the standard `public static void main(String[])` entry point.

```kotlin
main {
	// emit bytecode
	return_()
}
```

The body is responsible for emitting whatever instructions are needed, including the terminating return instruction.

## Bytecode helpers inside method bodies

The following helpers are available inside `CodeBuilder` blocks.

### Constant loading: `ldc(constant)`

Loads a constant onto the operand stack.

Supported types:

- `String`
- `Int`
- `Long`
- `Float`
- `Double`

```kotlin
ldc("Hello")
ldc(42)
ldc(3.14)
```

Unsupported constant types throw `IllegalArgumentException`.

### Field access

#### `getfield(ref)`
Reads a field using a `FieldRef`.

#### `getstatic(ref)`
Reads a static field using a `FieldRef`.

#### `getstatic<T>(name)`
Reads a public static field by reflection using `Class.getField`.

> The reflection-based overload expects a public field. Use the `FieldRef` overload when you already have a field reference.

#### `getThisField(ref)`
Convenience helper that emits `aload(0)` followed by `getfield(ref)`.

#### `putfield(ref)`
Writes the current stack value into the referenced field.

#### `putstatic(ref)`
Writes the current stack value into the referenced static field.

#### `putThisField(ref, value)`
Convenience helper that emits:

```kotlin
aload(0)
ldc(value)
putfield(ref)
```

This overload is useful for storing a constant value into an instance field.

#### `putThisField(ref, param)`
Convenience helper that emits:

```kotlin
aload(0)
aload(param)
putfield(ref)
```

This is useful when the value comes from a local variable or method parameter.

### Method invocation

#### Reflection-based method reference helpers

These helpers resolve members through reflection and build the corresponding method reference for bytecode emission:

- `buildMethodRefEntry<T>(name, vararg args)`
- `invokevirtual<T>(name, vararg args)`
- `invokestatic<T>(name, vararg args)`
- `invokespecial<T>(name = INIT_NAME, vararg args)`

Notes:

- `buildMethodRefEntry` and the reflective invocation helpers use `Class.getMethod`, so they resolve **public** methods only.
- `invokespecial<T>(INIT_NAME, ...)` resolves a constructor with `Class.getConstructor`.
- If you need to invoke a private or same-class member, use the `MethodRef`-based overloads.

Examples:

```kotlin
invokevirtual<String>("concat", String::class)
invokestatic<System>("currentTimeMillis")
invokespecial<MyType>(INIT_NAME)
```

#### `invokevirtual(methodRef)`
Invokes a method using a prebuilt `MethodRef`.

#### `invokestatic(methodRef)`
Invokes a static method using a prebuilt `MethodRef`.

#### `invokespecial(methodRef)`
Invokes a constructor or special method using a prebuilt `MethodRef`.

#### `invokeMethod(methodRef)`
Dispatches to the correct invocation opcode based on the `MethodRef` metadata:

- static members -> `invokestatic`
- constructors and private methods -> `invokespecial`
- everything else -> `invokevirtual`

### Control flow

#### `defaultConstructor()`
Emits a direct call to the superclass no-argument constructor.

This helper assumes the superclass has an accessible zero-argument constructor.

#### `whileLoop(condition, body)`
Builds a low-level `while` loop structure.

- `condition` receives the exit label and should jump to it when the loop should end
- `body` emits the loop body

Example:

```kotlin
whileLoop(
	condition = { end ->
		// emit condition and branch to end when false
	},
	body = {
		// loop body
	}
)
```

## Output and loading

The final result of the DSL is a `Klass` wrapper.

### `klass(): Klass`
Finalizes the collected class definition and returns the wrapper.

### `Klass`

`Klass` holds the generated bytes and provides the common ways to consume them.

#### `bytes`
The raw generated bytecode as a `ByteArray`.

#### `writeAndGetClass<T>()`
Writes the generated bytes to disk and loads the class as a `KClass<T>`.

Use a `T` that is assignable from the generated class.

#### `writeAndGetInstance<T>()`
Writes the generated bytes to disk and creates a new instance of the generated class using `createInstance()`.

This requires a public no-argument constructor.

#### `load<T>()`
Loads the generated class without writing it to disk.

#### `writeToDisk()`
Writes the generated bytes to disk using the builder's class name.

## Complete example

```kotlin
import java.io.PrintStream
import kotlin.reflect.full.createInstance

interface Animal {
	fun speak(): String
	fun talk()
	fun setName(name: String)
}

fun main() {
	val dogClass = KlassFileBuilder.klass("Dog") {
		inherit<Animal>()

		val nameField = field<String>("name")

		constructor {
			defaultConstructor()
			putThisField(nameField, "Jerry")
			return_()
		}

		val speak = method("speak", returnType = CD_String) {
			ldc("Woof! My name is ")
			getThisField(nameField)
			invokevirtual<String>("concat", String::class)
			areturn()
		}

		method("setName", args = arrayOf(CD_String)) {
			putThisField(nameField, 1)
			return_()
		}

		method("talk") {
			getstatic<System>("out")
			invokeMethod(speak)
			invokevirtual<PrintStream>("println", String::class)
			return_()
		}
	}

	val dog = dogClass.writeAndGetClass<Animal>().createInstance()
	dog.talk()
	dog.setName("Buddy")
	println(dog.speak())
}
```

## Working with method references

Method builders return `MethodRef`, so you can keep them and reuse them later.

```kotlin
val greet = method("greet", returnType = CD_String) {
	ldc("Hello, World!")
	areturn()
}

method("sayHello") {
	invokeMethod(greet)
	return_()
}
```
